DVL — Zentrale Datenerfassung

A centralized data management system for virtual media, studio operations, and lecture recording workflows across 7 integrated modules

Role Designer & Developer
Year 2025
Status Internal

About this Project

A centralized data management system I designed and built for one of Germany's largest private universities. The platform — "Zentrale Datenerfassung Virtuelle Medien & Studios" — unifies 7 integrated modules: VL-AZ (lecture recording processing), CMP (content & media production), StM (studio management), SP (study platforms), QvL (qualification for virtual teaching), KI (AI & automation), and VMS (media & studios). It replaced fragmented spreadsheet workflows with a single, real-time dashboard.

The system manages hundreds of weekly lecture recordings end-to-end — from Zoom permission assignment and Panopto upload tracking to Moodle course linking and escalation workflows. It features a full CRM with detailed views for each event (Veranstaltung, Lehrender, Kurs-Info, CMP & STM details), studio supervisor feedback integration, documentation tracking, and automated notifications for missing recordings or expired deadlines.

Tech Stack

Django / Python HTML / CSS / JS Panopto API Moodle Integration Zoom API Power BI

Key Features

  • 7 integrated modules (VL-AZ, CMP, StM, SP, QvL, KI, VMS)
  • Full CRM with detailed event & lecturer views
  • Real-time recording & upload status tracking
  • Panopto, Moodle & Zoom integration
  • Escalation workflows & deadline tracking
  • Studio feedback loop & documentation
